# Daily Routine SOP
**Standard Operating Procedure for Personal Daily Workflow**

**Last Updated:** 2026-03-21
**Owner:** [Your Name] ([Your Company] Workspace)
**Status:** Active
**Cadence:** Daily (Monday-Friday recommended, adjustable for weekend catch-up)

---

## Purpose
This SOP defines the daily workflow for managing email, calendar, tasks, and office coordination across the AI employee workspace. It ensures consistent communication, delegation, and visibility across all roles and prevents bottlenecks or missed escalations.

---

## Daily Rhythm Overview

| Time Block | Activity | Duration | Owner |
|-----------|----------|----------|-------|
| 8:00-8:30 AM | Morning Setup & Triage | 30 min | You |
| 8:30-12:00 PM | Active Work & Delegation | 3.5 hrs | You + Team |
| 12:00-1:00 PM | Lunch / Break | 1 hr | - |
| 1:00-4:00 PM | Follow-up & Routing | 3 hrs | You + Team |
| 4:00-5:00 PM | Daily Close & Prep | 1 hr | You |

**Total Administrative Time:** ~1.5 hrs/day (varies by workload)

---

## MORNING ROUTINE (8:00–8:30 AM)

### 1. System Check-in
- [ ] Open OFFICE_OPS Dashboard (`/OFFICE_OPS/dashboard.html`)
- [ ] Review today's date and capacity status (auto-populated from ROSTER.md)
- [ ] Note any ongoing incidents or escalations from `/OFFICE_OPS/incidents/`

### 2. Email Triage
- [ ] Open inbox; set dedicated triage time (no interruptions)
- [ ] Scan for urgent flags: **ESCALATE**, **DECISION**, **BLOCKER**, **URGENT**
- [ ] Sort emails into temporary buckets:
  - **Now:** Immediate responses or routing (5–10 emails expected)
  - **Soon:** Delegation or follow-up (10–15 emails)
  - **Later:** Reference or archive (remainder)
- [ ] **Document:** Log triage results in `/OFFICE_OPS/reporting/daily/[DATE]-email-triage.md`
  - Format: Summary of urgent items, who they route to, any blockers
  - *Rationale:* Provides audit trail and identifies patterns

### 3. Calendar Review
- [ ] Review today's calendar for:
  - Back-to-back meetings (flag overload)
  - Any external/client calls (prep time needed)
  - Decision points or approvals needed
- [ ] Block 30 min for "admin time" if not already scheduled
- [ ] Note any calendar holds that need routing:
  - *Example:* Executive Assistant may need to join calls with C-suite
  - *Example:* Legal Counsel may need to review vendor contract discussions

### 4. Task Inventory
- [ ] Check task management system (email, calendar, shared docs, or Notion)
- [ ] Identify today's 3–5 critical tasks
- [ ] Flag any dependencies on other team members (see Escalation Routing below)

---

## ACTIVE WORK PERIOD (8:30 AM–12:00 PM)

### 1. Process Email Triage Backlog
For each "Now" bucket item:

**Is it for you personally?**
- [ ] If yes → respond within 1 hour if possible
- [ ] If no → route to appropriate employee (see Escalation Matrix below)

**Is it a question for the office?**
- [ ] Identify the primary owner (check role descriptions)
- [ ] Draft a brief email routing to that person
- [ ] CC relevant supporting roles (see COORDINATION_MATRIX.md)
- [ ] Mark original email as "delegated" or archived

**Is it a blocker or escalation?**
- [ ] If it's a business decision → flag for your review before noon
- [ ] If it's a conflict between two roles → review COORDINATION_MATRIX.md for precedent
- [ ] If it requires external approval → route to Legal Counsel or Briefing Officer

### 2. Delegation & Role Assignment
Use this routing matrix for common scenarios:

| Request Type | Primary Owner | Supporting Roles |
|--------------|---------------|-----------------|
| **Sales inquiry** | Account Executive or SDR | Account Manager (if existing customer) |
| **Customer issue** | Support Specialist | Customer Success Manager (if health risk) |
| **Content request** | Content Writer or Marketing Coordinator | Briefing Officer (if strategy input needed) |
| **Contract/NDA** | Legal Counsel | Executive Assistant (routing), Finance (if budget-related) |
| **Hiring/People** | HR Coordinator or Recruiter | Operations Manager (process review) |
| **Data/Analytics** | Data Analyst | Operations Manager (if process impact) |
| **Product/Technical** | Product Manager | Project Manager (if timeline required) |
| **Leadership brief** | Briefing Officer | Executive Assistant (scheduling) |
| **Urgent escalation** | YOU → determine primary owner | Operations Manager or Executive Assistant (coordination) |

*Full routing precedent is in `/OFFICE_OPS/coordination/COORDINATION_MATRIX.md`*

### 3. Status Check-ins (Async)
- [ ] Scan for any "awaiting response" items from yesterday
- [ ] Check if dependent team members have sent updates
- [ ] Unblock any stalled work (30-second scan)

---

## LUNCH / BREAK (12:00–1:00 PM)
- [ ] Step away from desk
- [ ] Return emails marked "urgent" only if absolutely critical
- [ ] Prepare for afternoon follow-up period

---

## FOLLOW-UP & COORDINATION PERIOD (1:00–4:00 PM)

### 1. Monitor Delegated Work
- [ ] Check for responses from team members on items you routed
- [ ] If no response after 2 hours, send 1-line follow-up reminder
- [ ] Escalate to Operations Manager if blocking customer/revenue work

### 2. Ad-Hoc Requests
- [ ] Handle new email arrivals with same triage logic as morning
- [ ] Prioritize:
  1. Revenue-blocking items (Sales, Account Management)
  2. Customer-critical items (Support, Customer Success)
  3. Operational items (everything else)

### 3. Cross-Role Coordination
- [ ] If multiple roles need to align (e.g., Product + Engineering + Marketing launch):
  - [ ] Create a brief Slack/email thread summarizing:
    - Decision/request
    - Who needs to weigh in (and by when)
    - Deadline for input
  - [ ] Reference existing playbook if available (check `/OFFICE_OPS/projects/` or `/OFFICE_OPS/training/`)

### 4. Incident Monitoring
- [ ] Check `/OFFICE_OPS/incidents/` for any open items
- [ ] If incident is >4 hours old without resolution, route to Operations Manager
- [ ] Update incident log if you learn new info

---

## DAILY CLOSE (4:00–5:00 PM)

### 1. Email Wrap-up
- [ ] Process remaining "Soon" bucket (10 min)
- [ ] Move "Later" items to archive or reference folder
- [ ] Inbox should reach <5 unread items by 5 PM (or archive them for next day)

### 2. Task Close-out
- [ ] Mark completed items as done in task system
- [ ] Identify any tasks rolling to tomorrow (reschedule or assign)
- [ ] Highlight any tasks at risk of missing deadline

### 3. Update Daily Reporting Log
- [ ] Update `/OFFICE_OPS/reporting/daily/[TODAY]-summary.md` with:
  - **Emails processed:** count + any escalations
  - **Delegated items:** who + what + deadline
  - **Blockers:** list any items stuck or awaiting escalation
  - **Tomorrow's focus:** 3–5 priorities
- [ ] *Rationale:* Creates continuity if you're unavailable; helps Operations Manager track workflow

### 4. Prepare Tomorrow
- [ ] Flag any items due tomorrow (calendar holds, approvals, responses)
- [ ] Brief Executive Assistant on any calendar changes (if applicable)
- [ ] Close all windows; reset workspace for next morning

## ESCALATION MATRIX & DECISION RULES

### When to Escalate to Operations Manager
- [ ] Email/request sits with an employee >4 hours with no movement
- [ ] Conflict between two roles on priority/ownership
- [ ] A non-assigned request comes in (unclear owner)
- [ ] Customer is waiting on internal alignment
- [ ] Resource allocation question (who has capacity?)

### When to Escalate to Executive Assistant
- [ ] C-level needs scheduling or briefing
- [ ] Sensitive/confidential communication
- [ ] External event coordination
- [ ] Leadership-level decision needed before delegation

### When to Escalate to Legal Counsel
- [ ] Any external contract or agreement
- [ ] Regulatory/compliance question
- [ ] Employee dispute or sensitive HR matter
- [ ] Customer dispute or liability concern

### When to Escalate to You (Stop & Decide)
- [ ] Revenue-blocking decision (ask: does this risk a deal?)
- [ ] Cross-customer conflict (competing asks)
- [ ] Process change request (someone asking to change SOP)
- [ ] Resource reallocation (is someone overloaded?)

## Quality Checkpoints (Weekly Review)

Every Friday at 4:30 PM, review this week's daily logs:
- [ ] Were emails triaged within 1 hour of arrival?
- [ ] Did any item sit with an employee >4 hours without movement?
- [ ] Was there any miscommunication about role boundaries?
- [ ] Did any critical item fall through the cracks?

**Action:** If any item answers "yes," note the pattern and decide:
- Do we need to update this SOP?
- Do we need to clarify a role's responsibilities?
- Do we need to rebalance capacity (check ROSTER.md)?

---

## Emergency / Overload Protocol

**If email volume exceeds 100 items in a day:**
1. [ ] Acknowledge receipt of all items (don't lose them)
2. [ ] Triage ONLY "Now" bucket items (5–10 items)
3. [ ] For "Soon" bucket: assign each to an owner with 24-hour response window
4. [ ] For "Later" bucket: batch process next morning
5. [ ] Alert Operations Manager that day was overloaded (pattern recognition)

**If you're unavailable (sick, OOO, etc.):**
1. [ ] Delegate to Operations Manager or Executive Assistant
2. [ ] Leave clear notes on any in-flight routing
3. [ ] Update `/OFFICE_OPS/incidents/` if customer-facing items are pending
