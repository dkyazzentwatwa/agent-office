# Incidents Folder Intro

## Purpose
Use this folder to track problems, failures, conflicts, and operational exceptions that need visibility.

## What goes here
- Incident logs
- Resolution notes
- Follow-up actions
- Gap or failure records

## How to use this folder
Add entries when something breaks, stalls, or creates risk. Keep records factual and focused on what happened, impact, owner, and next action.

## Notes for AI agents
Use this folder when diagnosing repeat issues or checking whether a problem is already known. Escalate quickly if an incident affects multiple teams or critical workflows.
