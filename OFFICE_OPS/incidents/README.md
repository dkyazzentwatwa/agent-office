# Incidents

This folder is optional evidence storage for blockers, failures, and repeat issues.

## What belongs here

- operational blockers
- customer- or revenue-impacting failures
- sensitive escalation notes
- repeat routing or handoff problems
- follow-up actions after an issue is resolved

## Use this folder when

- something needs visibility beyond a daily log
- an issue may repeat
- the office needs a durable record of what happened and what changed

If there are no incidents, this folder can stay empty except for this README.
