# First User Guide

## What Agent Office is

Agent Office is a shared AI workspace made up of role-based employees. Each employee has a defined job, style, scope, and coordination pattern. Instead of treating the system like one generic assistant, you get better results by working with the right employee or combination of employees.

This guide is for both:
- people using this repo directly
- people working through ChatGPT or Claude projects that use these employee files

## How to work with the office

There are three simple ways to use Agent Office:

1. Ask for the best employee
If you are not sure who should handle something, ask the office to route it.

2. Ask for a specific employee
If you already know the role you want, name that employee directly.

3. Ask for coordination
If the work crosses departments, ask for a multi-role plan instead of forcing one employee to do everything.

## How routing works

The office should route work based on role fit first, then coordination needs, then capacity.

Examples:
- Product questions should go to Product Manager
- Process or workflow design should go to Operations Manager
- Reporting and trend analysis should go to Data Analyst
- Contracts or policy review should go to Legal Counsel
- Multi-step cross-functional work should usually be coordinated, not handled by one employee alone

## When to name a role directly

Name a role when:
- you want a response in that employee's voice
- you want a role-specific opinion
- you already know who owns the task
- you want to compare one employee against another

Example prompts:
- "Answer as the Product Manager."
- "What would Legal Counsel flag here?"
- "Have the Operations Manager turn this into a workflow."

## When to ask for routing

Ask for routing when:
- the problem is broad
- ownership is unclear
- you want to know who should lead
- more than one employee may be involved

Example prompts:
- "Who should handle this?"
- "Which employee owns this request?"
- "Who should lead this launch, and who supports them?"

## When to ask for coordination

Ask for coordination when the work needs handoffs between roles.

Common cases:
- product launch planning
- campaign execution
- onboarding and retention work
- compliance or risk review
- process redesign

Example prompts:
- "Coordinate a launch plan using Product, Project, and Marketing."
- "Map the handoff between Sales, Customer Success, and Support."
- "Who needs to be involved in this compliance review?"

## Common ways to use Agent Office

### Get a role-based answer
Ask what a specific employee would do.

### Compare roles
Ask for the difference between similar employees such as Product Manager vs Project Manager, or Support Specialist vs Customer Success Manager.

### Build a workflow
Ask the office to turn a task into a step-by-step operating plan with owners and handoffs.

### Review a draft or decision
Ask the most relevant employee to review a message, plan, contract, summary, or workflow.

### Route incoming work
Paste a request, task, or issue and ask who should own it.

## How to get better outputs

- Give the office the real task, not just a topic
- Name the employee if you already know the best role
- Ask for a format if you want something specific, such as a checklist, plan, draft, summary, or review
- If the task crosses functions, say that explicitly
- If you want practical output, ask for deliverables and next steps

## Copy-ready prompts

- "Who should handle this, and why?"
- "Answer as the Executive Assistant and draft a leadership update."
- "Compare Product Manager and Project Manager for this situation."
- "Turn this into a cross-functional workflow with owners and handoffs."
- "Which employee should lead this, and who should support them?"
- "Review this plan as Legal Counsel."
- "Route this request to the right office team."

## Good default mindset

Treat Agent Office like a staffed team:
- ask for the right specialist
- let the office coordinate when needed
- use role comparisons when ownership is unclear
- ask for outputs you can actually use

The more clearly you frame the task, owner, and desired outcome, the better the office performs.
